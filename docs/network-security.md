# 集群网络暴露收口方案

为保障 StreamLake-Binance 系统的底层数据安全，防止端口被公网扫描器利用，特制定以下安全组规则和网络隔离规范。

## 1. 端口隔离规范

| 组件 | 端口 | 开放范围 | 说明 |
| --- | --- | --- | --- |
| **Kafka Broker** | `9092` | 仅内网和特定管理 IP | **严禁对外开放**，仅供 Flink 和 Collector 内部通信。 |
| **Flink WebUI** | `8081` | 仅 VPN 或特定管理 IP | **严禁公网开放**，防止未经授权提交恶意 Jar 包执行任意代码。 |
| **Doris FE MySQL 协议** | `9030` | 仅内网和特定管理 IP | **严禁公网开放**。 |
| **Doris FE HTTP** | `8030` | 仅内网和特定管理 IP | **严禁公网开放**。 |
| **MySQL** | `3306` | 仅内网和特定管理 IP | **严禁公网开放**。 |
| **Redis** | `6379` | 仅内网 | **原则上不允许公网开放**，即使配置了密码。 |
| **Spring API** | `8080` | 内网或反向代理后暴露 | 若需对外提供服务，必须前置 Nginx/API 网关进行反向代理、限流与鉴权。 |
| **AI Engine** | `8000` | 仅内网 | 供 Spring API 和本地调度脚本内部调用，**严禁公网开放**。 |

## 2. 云控制台手工规则要求 (TODO)

管理员需在阿里云/AWS/腾讯云控制台中，检查相关的安全组 (Security Group) 配置：
1. 删除 `0.0.0.0/0` 针对 `9030`, `3306`, `6379`, `8081`, `9092` 的放行规则。
2. 配置内网网段 (例如 `172.16.0.0/16` 或 `192.168.0.0/16`) 允许互通。
3. 若需要远程排查，请将公司出口公网 IP 添加到白名单中，而非全网放通。
