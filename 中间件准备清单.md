### Step 2：创建 `risk_rules` 表（Job-3 风控广播用）

**字段来源推断**（来自 RiskControlJob.java 第 145 行）：
```java
"SELECT id, symbol, max_single_qty FROM risk_rules WHERE enabled = 1"
```

```sql
CREATE TABLE IF NOT EXISTS risk_rules (
    id              INT             NOT NULL AUTO_INCREMENT COMMENT '规则ID',
    symbol          VARCHAR(20)     NOT NULL                COMMENT '交易对，如 BTCUSDT；* 表示兜底规则',
    max_single_qty  DECIMAL(30, 8)  NOT NULL                COMMENT '单笔最大数量阈值，超过则触发风控',
    enabled         TINYINT(1)      NOT NULL DEFAULT 1      COMMENT '是否启用：1=启用，0=禁用',
    remark          VARCHAR(200)    DEFAULT NULL            COMMENT '备注说明',
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uq_symbol (symbol)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='风控规则表';
```

**插入初始规则数据**：
```sql
-- 兜底规则：所有交易对，单笔超过 100 BTC/1000 ETH 数量等价均触发
-- qty 是币的数量（不是 USDT），请根据实际需要调整
INSERT INTO risk_rules (symbol, max_single_qty, enabled, remark) VALUES
('*',       100.00000000, 1, '通配兜底规则：单笔数量超100触发'),
('BTCUSDT',  50.00000000, 1, 'BTC：单笔超50个触发'),
('ETHUSDT',  500.00000000, 1, 'ETH：单笔超500个触发'),
('BNBUSDT',  1000.00000000, 1, 'BNB：单笔超1000个触发'),
('SOLUSDT',  5000.00000000, 1, 'SOL：单笔超5000个触发');
```

> **热更新说明**：Job-3 每 30 秒重新 SELECT 一次，修改这张表的数据后无需重启作业，下一个轮询周期（最长 30 秒）自动生效。

### Step 3：创建 `whale_thresholds` 表（Job-2 CEP 巨鲸用）

**字段来源推断**（来自 WhaleCepJob.java 第 162 行）：
```java
"SELECT symbol, threshold_quote FROM whale_thresholds WHERE enabled = 1"
```

```sql
CREATE TABLE IF NOT EXISTS whale_thresholds (
    id               INT             NOT NULL AUTO_INCREMENT COMMENT '主键',
    symbol           VARCHAR(20)     NOT NULL                COMMENT '交易对，如 BTCUSDT；* 表示默认阈值',
    threshold_quote  DECIMAL(30, 2)  NOT NULL                COMMENT '巨鲸成交额阈值（USDT），单笔超过此值算大单',
    enabled          TINYINT(1)      NOT NULL DEFAULT 1      COMMENT '是否启用：1=启用，0=禁用',
    remark           VARCHAR(200)    DEFAULT NULL            COMMENT '备注说明',
    created_at       DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at       DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uq_symbol (symbol)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='巨鲸告警阈值配置表';
```

**插入初始阈值数据**（与代码内置默认值对应）：
```sql
-- 代码内置默认：BTC=50000，ETH=20000，其他=10000
-- 向 MySQL 写入后会覆盖内置值
INSERT INTO whale_thresholds (symbol, threshold_quote, enabled, remark) VALUES
('*',       10000.00, 1, '通配默认：单笔成交额超1万USDT'),
('BTCUSDT', 50000.00, 1, 'BTC：单笔超5万USDT'),
('ETHUSDT', 20000.00, 1, 'ETH：单笔超2万USDT'),
('BNBUSDT', 15000.00, 1, 'BNB：单笔超1.5万USDT'),
('SOLUSDT', 10000.00, 1, 'SOL：单笔超1万USDT'),
('XRPUSDT', 8000.00,  1, 'XRP：单笔超8千USDT'),
('DOGEUSDT',5000.00,  1, 'DOGE：单笔超5千USDT');
```

> **注意**：Job-2（WhaleCepJob）只在**作业启动时**加载一次 MySQL 阈值，不会动态热更新。
> 如果修改了表数据，需要重启作业才能生效。

| 作业 | 用途 | Key 格式 | TTL |
|---|---|---|---|
| **Job-3 RiskControlJob** | 触发风控后写黑名单 | `risk:blacklist:{SYMBOL}` | 86400 秒（24小时） |

**写入示例**（来自 RedisBlacklistSink.java）：


---

### 方案 A：apt 直接安装（推荐，简单）

```bash
# 在 master-01 上执行
sudo apt-get update
sudo apt-get install -y redis-server

# 验证版本
redis-server --version  
# 期望：Redis server v=7.x.x
```

> Ubuntu 22.04 apt 仓库的 Redis 版本约为 7.0.x，满足规划要求的 7.2.x。
> 若需精确 7.2.x，使用方案 B。

---

### 方案 B：Docker 安装（精确版本控制）

```bash
docker run -d \
  --name redis \
  --restart unless-stopped \
  -p 6379:6379 \
  -v /opt/redis/data:/data \
  redis:7.2-alpine \
  redis-server --appendonly yes --bind 0.0.0.0
```

---

### Step：修改 Redis 配置（apt 安装后）

编辑 `/etc/redis/redis.conf`，关键修改项：

```bash
# 1. 允许所有内网地址访问（默认只监听 127.0.0.1）
bind 0.0.0.0

# 2. 关闭保护模式（内网使用，否则非本机连接会被拒绝）
protected-mode no

# 3. 开启 AOF 持久化（黑名单数据重启后不丢）
appendonly yes
appendfilename "appendonly.aof"

# 4. 配置最大内存（防止占满系统内存）
maxmemory 512mb
maxmemory-policy allkeys-lru
```

```bash
# 重启 Redis 使配置生效
sudo systemctl restart redis-server
sudo systemctl enable redis-server

# 验证
redis-cli ping
# 期望：PONG
```

---

### Step：验证 Redis 可被 Flink 访问

```bash
# 从 worker-01 / worker-02 上测试连通性

# 手动模拟写入一条黑名单（测试用）

redis-cli -h 192.168.1.10 ttl "risk:blacklist:BTCUSDT"
```

## 三、准备顺序建议（避免踩坑）

```
1. MySQL 建库建表 + 插入初始数据   ← 必须在提交 Job-2/Job-3 之前完成
2. Redis 安装 + 修改 bind 配置      ← 必须在提交 Job-3 之前完成
3. 验证两者从 worker 节点均可访问   ← 防止 Flink TaskManager 连不上
4. 再提交 Flink 三个 JAR
```
